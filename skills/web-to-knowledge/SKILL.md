# Web to Knowledge

将网页内容精炼、总结、补充，整理为结构化的个人知识文档。

## When to Activate

- 用户提供 URL，要求总结/提炼/整理成笔记
- 用户说"总结这个网页"、"把这篇文章整理成笔记"、"提取关键知识"
- 用户说"知识卡片"、"网页转文档"、"网页笔记"
- 用户已有网页文本内容，要求精炼重构

## 与 deep-research 的区别

| 维度 | web-to-knowledge | deep-research |
|------|-----------------|---------------|
| 输入 | 1-3 个网页 URL 或文本 | 需要广泛调研的主题 |
| 目标 | 知识提取与重构 | 多源综合研究 |
| 输出 | 个人知识文档 | 研究报告（含引用） |
| 侧重 | 精炼、补充、内化 | 搜索、交叉验证、引用 |

## Workflow

### Step 1: 获取内容

用户提供 URL 时，使用 `fetch_content` 抓取网页内容：

```
fetch_content(url: "<user_provided_url>", query: "提取文章主体内容、核心观点、关键细节")
```

用户提供纯文本时，直接进入 Step 2。

**处理多 URL**：用户给多个 URL 时，并行抓取，合并处理。

### Step 2: 分析与结构化

快速扫描内容，识别：
- **核心主题**：文章在讲什么
- **知识密度**：哪些段落信息密度高（保留），哪些是水分/铺垫（压缩）
- **结构层次**：天然的分节/逻辑链
- **知识盲区**：读者可能缺少的前置知识（需要补充）

### Step 3: 精炼与重构

按以下原则处理原始内容：

**精炼（做减法）**：
- 删除废话、过渡句、重复论述
- 将冗长描述压缩为精确表达
- 合并同类信息

**总结（做归纳）**：
- 每个章节/段落提炼 1-3 个核心要点
- 用自己的话重新表述，而非复制原文
- 添加章节小结

**补充（做加法）**：
- 补充原文隐含但未明说的前置知识
- 添加必要的背景信息帮助理解
- 补充实际应用场景或注意事项
- 标注原文未覆盖的相关知识缺口（用 > 💡 标记）

### Step 4: 输出知识文档

使用 [knowledge-template.md](knowledge-template.md) 模板生成 Markdown 文件。

**输出规范**：
- 文件名：`{主题关键词}.md`，由用户指定或从内容推断
- 保存位置：用户指定目录，默认为当前工作空间根目录
- 生成后告知用户文件路径

### Step 5: 质量自检

交付前快速检查：
- [ ] 知识点是否完整（无重大遗漏）
- [ ] 补充内容是否标注（区分原文 vs 补充）
- [ ] 结构是否清晰（3 层以内可定位任何知识点）
- [ ] 是否有不理解的概念需要用户确认

## 精炼技巧速查

| 场景 | 处理方式 |
|------|----------|
| 教程/How-to | 提取步骤 + 关键参数，删除解释性文字 |
| 技术博客 | 提取方案 + 代码片段 + 适用条件，删除动机描述 |
| 官方文档 | 提取 API 签名 + 约束 + 示例，删除营销语言 |
| 观点文章 | 提取论点 + 论据 + 逻辑链，删除情感表达 |
| 新闻资讯 | 提取事实 + 影响 + 背景，删除记者评论 |
| 论文/研究 | 提取问题 + 方法 + 结论 + 局限，删除文献综述 |

## 注意事项

1. **忠于原意**：精炼不等于曲解，核心观点必须忠实原文
2. **标注来源**：文档头部记录来源 URL 和抓取日期
3. **区分层次**：用 `> 💡` 标记补充内容，与原文精炼内容区分
4. **适度补充**：补充是帮助理解，不是写新文章；控制在原文篇幅 10-20%
5. **中文优先**：输出文档使用中文，技术术语保留英文原文
