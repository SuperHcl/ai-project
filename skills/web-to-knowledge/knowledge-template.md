# 知识文档模板

以下为知识文档的标准输出格式，根据内容类型选择对应模板。

---

## 通用知识文档模板

```markdown
# {标题}

> 来源：{URL}
> 整理日期：{YYYY-MM-DD}
> 原文作者：{作者}（如可识别）

## 概述

{3-5 句话概述，回答：这是什么？解决什么问题？核心结论是什么？}

---

## 核心内容

### {章节1标题}

{精炼后的知识点}

- 要点一
- 要点二
- 要点三

> 💡 补充：{原文未提及但理解此章节所需的前置知识}

### {章节2标题}

{精炼后的知识点}

**关键概念**：
| 概念 | 说明 |
|------|------|
| {概念A} | {一句话解释} |
| {概念B} | {一句话解释} |

### {章节N标题}

{按需添加章节，保持 2-4 个主章节}

---

## 要点速记

1. {最核心的一个知识点}
2. {第二核心}
3. {第三核心}
4. {第四核心}
5. {第五核心}

---

## 应用笔记

> 💡 以下为补充内容，非原文所有

**适用场景**：{该知识在什么场景下有用}

**注意事项**：{使用时需要特别注意的点}

**延伸方向**：{值得进一步探索的相关主题}
```

## 核心机制

{精炼技术原理，配关键代码片段或架构图}

```{lang}
// 仅保留最关键的代码片段，删除样板代码
```

## 关键参数 / API

| 参数/API | 说明 | 默认值 | 注意事项 |
|----------|------|--------|----------|
| | | | |

## 使用步骤

1. {步骤一}
2. {步骤二}
3. {步骤三}

> 💡 补充：{原文未提到的坑点或最佳实践}

## 适用条件

- ✅ 适用：{场景A}、{场景B}

```

## 观点/分析文章模板

适用于：行业分析、观点评论、趋势预测

```markdown
# {主题}

> 来源：{URL}
> 整理日期：{YYYY-MM-DD}

## 核心论点

{作者的核心主张，1-2 句话}

## 论据链

| # | 论点 | 支撑论据 | 逻辑强度 |
|---|------|----------|----------|
| 1 | {分论点} | {关键数据/事实} | 强/中/弱 |
| 2 | | | |

## 关键数据

- {数据1}：{数值}（{来源}）
- {数据2}：{数值}（{来源}）

> 💡 补充视角：{原文未考虑的维度或反面观点}

## 个人判断

> 💡 以下为补充，非原文观点

- 采信度：高/中/低（{原因}）
- 与已知知识的冲突：{如有}
- 行动启示：{如有}
```

## 模板选择指南

| 内容类型 | 推荐模板 | 识别特征 |
|----------|----------|----------|
| 技术教程/文档 | 技术文档精炼模板 | 有代码、步骤、参数说明 |
| 行业分析/观点 | 观点/分析文章模板 | 有论点、论据、数据 |
| 综合知识/概念 | 通用知识文档模板 | 通用，无明确技术或观点特征 |
| 混合型 | 以通用模板为基础，按需嵌入技术/观点子模板 | 多种特征混合 |

**灵活使用**：模板是框架不是枷锁，根据实际内容调整章节。核心原则是：概述 → 核心内容 → 速记 → 应用笔记。
